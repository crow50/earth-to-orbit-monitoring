Guidelines for contributing
- Thank you for contributing! Before submitting changes, please read AGENTS.md in the repository root for information about agent roles and workspace conventions.
- Keep changes small and focused. Open issues for larger work and reference them from pull requests.
- Follow the project's git branch naming and commit message conventions. When in doubt, model your PR and messages on existing commits.
- Run tests and linters (if present) before creating a pull request.

If you have questions about contributing or need help, open an issue or reach out to the maintainers.
